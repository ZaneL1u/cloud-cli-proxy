<objective>
修复 AuthResponse.Status 字段无法解析 JSON 数字类型的问题。

当前登录报错：`json: cannot unmarshal number into Go struct field AuthResponse.status of type string`

原因：某些 gateway 响应中 `status` 字段返回的是 JSON 数字（如 `{"status": 1}`），而当前 `AuthResponse.Status` 是 `string` 类型，标准库 `json.Unmarshal` 拒绝将数字反序列化到字符串字段。

输出：
- `Status` 字段改为自定义类型，实现 `json.Unmarshaler` 接口，同时兼容数字和字符串两种 JSON 类型
- 补充测试覆盖数字 status 场景
- 保持所有现有调用点零破坏（`Status == "ready"` 等比较逻辑不变）
</objective>

       ```go
       // Status 兼容 JSON 字符串和数字两种形态的 status 字段。
       type Status string

       func (s *Status) UnmarshalJSON(data []byte) error {
           // 先尝试按字符串解析（带引号）
           var str string
           if err := json.Unmarshal(data, &str); err == nil {
               *s = Status(str)
               return nil
           }
           // 回退到数字解析，再转字符串
           var num int64
           if err := json.Unmarshal(data, &num); err == nil {
               *s = Status(strconv.FormatInt(num, 10))
               return nil
           }
           return fmt.Errorf("status 必须是字符串或数字， got: %s", string(data))
       }

       func (s Status) String() string {
           return string(s)
       }
       ```
       注意：需要导入 `strconv` 包。
