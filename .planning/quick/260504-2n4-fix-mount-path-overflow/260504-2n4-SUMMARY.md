# Quick 260504-2n4: 修复挂载路径在长路径下撑爆容器 Summary

修复创建主机对话框与主机挂载管理卡片在超长 source/target 路径下横向撑开的视觉溢出问题，根因为 flex 子项默认 `min-width: auto` 导致 `truncate` 失效。

## Diff 摘要

### `web/admin/src/components/hosts/create-host-dialog.tsx`

```diff
 {hostMounts.map((m, i) => (
   <div key={i} className="flex items-center gap-2 text-sm">
-    <span className="truncate font-mono" title={m.source}>{m.source}</span>
+    <span className="flex-1 min-w-0 truncate font-mono" title={m.source}>{m.source}</span>
     <span className="text-muted-foreground">-&gt;</span>
-    <span className="truncate font-mono" title={m.target}>{m.target}</span>
+    <span className="flex-1 min-w-0 truncate font-mono" title={m.target}>{m.target}</span>
```

-    <p className="mt-1 font-mono text-xs text-muted-foreground">
+    <p className="mt-1 truncate font-mono text-xs text-muted-foreground" title={m.target}>

## 根因（Root Cause）

Tailwind `truncate` = `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`。
在 flex 容器内，子项默认 `min-width: auto`，等于「至少能完整显示自己的内容宽度」。
所以再多 `truncate` 也无法收缩，反而把整个 flex 容器撑开。

修复手段：

1. 给 flex 子项追加 `min-w-0` 解除默认 min-width:auto 限制，允许收缩到 0。
2. 在 `create-host-dialog` 行布局里给两个路径 span 同时加 `flex-1`，让源、目标各占可用宽度的一半，删除按钮借助 `ml-auto` 仍稳定停在右侧。
3. 在 `mount-manager` 卡片中，外层 `<div>` 已有 `min-w-0 flex-1`，源 `<p>` 已 truncate，但目标 `<p>` 漏了 `truncate`，本次补齐；同时为两个 `<p>` 都补 `title`，hover 时可见完整路径，保持对称的可访问性。
