## 当前架构关键事实

1. **User 模型** (`internal/store/repository/models.go:18-32`): 已有 `Username`, `SSHPublicKey`, `SSHPrivateKey`, `SSHKeyType` 字段。`ShortID` 仍存在。

2. **HostSSHAuth 模型** (`models.go:52-65`): 当前有 `HostShortID string` 和 `Username string`（注释：关联用户的 username，用于连接容器内 sshd）。**改造后**：移除 `HostShortID`，保留 `Username`，新增 `SSHPrivateKey string`（控制面连接容器所需私钥）。

3. **Entry API** (`internal/controlplane/http/entry.go`):
   - `Script()` handler: URL 路径为 `/v1/entry/{shortId}/auth`，脚本里用 `shortID` 变量。
   - `Auth()` handler: 当前用 `shortID` 查 host 或 user，返回 `"ssh_user": hostShortID`。
   - **改造后**：URL 路径改为 `/v1/entry/{username}/auth`，`Auth()` 用 `username` 查 host，返回 `"ssh_user": auth.Username`。

4. **SSH proxy resolver** (`internal/sshproxy/resolver.go`):
   - `ResolveContainer(ctx, hostShortID, password)` — 按 shortID 查 host。
   - `ResolveContainerByPublicKey(ctx, hostShortID, clientKey)` — 同上。
   - `resolveTarget()` 返回 `ContainerTarget{Addr, User, Password}`，当前 `User` 为空（靠 proxy.go 默认值）。
   - **改造后**：参数改为 `username`；`resolveTarget` 返回 `User: auth.Username`；`Password` 改为 `SSHPrivateKey`（供 proxy 用私钥签名认证）。

5. **SSH proxy server** (`internal/sshproxy/proxy.go`):
   - `handleChannel()` 中 `targetUser` 为空时 fallback 到 `s.containerUser`（默认 workspace）。
   - `targetConfig.Auth` 当前用 `ssh.Password(pass)` + `ssh.PublicKeys(s.hostKey)`。
   - **改造后**：当 `targetPassword` 以 `-----BEGIN OPENSSH PRIVATE KEY-----` 或类似 PEM 开头时，解析为私钥并用 `ssh.PublicKeys(signer)` 认证；否则保留密码认证 fallback。

6. **cloud-claude CLI** (`cmd/cloud-claude/main.go`):
   - `initCmd` 有 `--short-id` flag，配置存 `cfg.ShortID`。
   - `runRoot` 用 `cfg.ShortID` 调 `AuthenticateAndWait()`。
   - **改造后**：`--short-id` 改为 `--username`，配置字段 `ShortID` 改为 `Username`，所有调用点替换。

7. **cloud-claude entry client** (`internal/cloudclaude/entry.go`):
   - `Authenticate(ctx, shortID, password)` 和 `AuthenticateAndWait(ctx, shortID, password, onWait)`。
   - **改造后**：参数名改为 `username`，内部 URL 构造路径同步改。

8. **Runtime ssh handoff** (`internal/runtime/tasks/ssh_handoff.go`):
   - metadata 里有 `"ssh_user": defaultUser`（当前是 request 里的值）。
   - **改造后**：确保 `defaultUser` 为 username（request.Username 或 fallback）。

9. **Bootstrap handoff** (`internal/controlplane/http/bootstrap_handoff.go`):
   - 返回 `"user": meta["ssh_user"]`。
   - **改造后**：确保 meta["ssh_user"] 为 username。

10. **Auth handler** (`internal/controlplane/http/auth_handler.go`):
    - 登录请求体有 `ShortID string `json:"short_id"`` 和 `Username string `json:"username"``。
    - 响应有 `"short_id": user.ShortID`。
    - **改造后**：请求体保留 `Username` 为主字段，移除 `ShortID`（或标记 deprecated）；响应移除 `short_id` 字段。

11. **测试覆盖**:
    - `entry_auth_test.go` 大量用 `shortID` 构造请求和 stub。
    - `sshproxy` 无现有单元测试（resolver 依赖 docker exec）。
    - **改造后**：entry_auth_test 全部改为 username；新增 resolver 的简单测试（stub repo）。

12. **数据库查询** (`queries.go`):
    - `GetHostByShortID` SQL 从 `hosts` 表按 `h.short_id = $1` JOIN `users` 取 `u.username`。
    - **改造后**：改为 `GetHostByUsername`，按 `u.username = $1` 查，同时 SELECT `u.ssh_private_key`。
