<objective>
修复 `ensureSeedAdmin` 凭据缺口：让它走完整的 entry_password + ed25519 SSH 密钥对生成流程，并写一行 `ssh_keys` 记录；对已存在但缺凭据的种子 admin 做幂等补齐。

**根因（来自 task_background）：** `internal/controlplane/app/app.go:199-226` 的 `ensureSeedAdmin` 仅调用 `CreateUserWithRole`（SQL 只插 username/password_hash/status/short_id/role），完全没生成 `entry_password / ssh_public_key / ssh_private_key`，也没写 `ssh_keys` 表。但 `internal/runtime/tasks/worker.go:219` 与 `:577` 硬约束 `EntryPassword != ""`，导致用 admin 创建的主机 worker 一律拒绝构造容器，前端永远停留在"创建中"。

**实现思路：**
1. 把 `internal/controlplane/http/admin_users.go` + `ssh_keys.go` 里的纯凭据生成函数（`generateRandomString` / `generateShortID` / `generateEntryPassword` / `generateLoginPassword` / `generateSSHKeyPair` / `computeFingerprint`）抽到新独立包 `internal/controlplane/credgen/`，导出为公开 API；http 包保留薄包装或直接改用 `credgen.*`。这是为了让 `app` 包能调用而不引入 `app → http` 的反向依赖（http 是 app 的下游）。
2. 重写 `ensureSeedAdmin`：拆出纯逻辑函数 `planSeedAdminCredentialFix(user repository.User)` 返回应该执行哪些写入动作，外壳函数负责调用 repo 与 credgen。fail-fast，任何步骤失败 return error 让控制面启动失败。
3. 幂等补齐：用 `GetUser`（已返回 entry_password / ssh_public_key / ssh_private_key 等列，见 queries.go:53-62）读现状，缺什么补什么；ssh_keys 表用 `ListSSHKeysByUserAndPurpose(userID, "inbound")` 检查是否已有 `label='auto-generated'` 行，存在则跳过插入。绝不 DELETE 老 ssh_keys 行。

Purpose: 解决用户上报的 P0 缺口（admin 创建主机卡"创建中"），同时把凭据生成模块做包级解耦，方便后续 `host-agent` 或脚本工具复用。

Output: 4 处代码改动 + 2 套单测 + 3 个独立 commit。

         pub, priv, err := credgen.GenerateSSHKeyPair("ed25519", username)               // fail-fast

         var user repository.User
         switch {
         case errors.Is(err, pgx.ErrNoRows):
             user, err = repo.CreateUserWithRole(ctx, repository.CreateUserWithRoleParams{

             })
             if err != nil { return fmt.Errorf("create seed admin: %w", err) }
             logger.Info("seed admin created", "short_id", username)
         case err == nil:
             user, err = repo.GetUser(ctx, existing.ID)
             if err != nil { return fmt.Errorf("get seed admin full row: %w", err) }
         default:
             return fmt.Errorf("check seed admin: %w", err)
         }

         keys, err := repo.ListSSHKeysByUserAndPurpose(ctx, user.ID, "inbound")
         if err != nil { return fmt.Errorf("list inbound ssh keys: %w", err) }
         hasInboundAutogen := false
         for _, k := range keys { if k.Label == "auto-generated" { hasInboundAutogen = true; break } }

         fix := planSeedAdminCredentialFix(user, hasInboundAutogen)
         if fix.NeedEntryPassword {
             if err := repo.UpdateUserEntryPassword(ctx, user.ID, entryPwd); err != nil { return fmt.Errorf("backfill seed admin entry password: %w", err) }
         }
         if fix.NeedSSHKeys {
             if err := repo.UpdateUserSSHKeys(ctx, user.ID, pub, priv, "ed25519"); err != nil { return fmt.Errorf("backfill seed admin ssh keys: %w", err) }

         if fix.NeedSSHKeysRow {

             }
         }

                 "entry_password", fix.NeedEntryPassword,
                 "ssh_keys", fix.NeedSSHKeys,
                 "ssh_keys_row", fix.NeedSSHKeysRow,
             )
         }
         return nil
