<task type="auto">
  <name>Task 1: 后端实现导出和导入 handler</name>
  <files>internal/controlplane/http/admin_hosts.go</files>
  <action>
在 `admin_hosts.go` 中 `AdminHostsHandler` 上新增两个方法：

1. `ExportConfig() nethttp.Handler`
   - 从 path 读取 `hostID`，查库验证主机存在且 `status == "running"`
   - 容器名：`"cloudproxy-" + hostID`
   - 执行：`docker exec -i containerName tar czf - -C /workspace .claude .claude.json .chrome-data -C /var/lib/claude-persist .`
   - 注意：tar 的 `-C` 选项用于切换目录，确保归档内路径正确；需要包含 `/var/lib/claude-persist/` 下的 `.claude` 和 `.cache/claude`
   - 设置响应头：`Content-Type: application/gzip`，`Content-Disposition: attachment; filename="host-{hostID}-config.tar.gz"`
   - 将 `cmd.Stdout` 直接 pipe 到 `w`（使用 `io.Copy`），避免内存中缓冲整个归档
   - 记录事件：`admin.host.config_exported`
   - 错误处理：非 running 返回 409；docker exec 失败返回 502；其他按现有模式

2. `ImportConfig() nethttp.Handler`
   - 从 path 读取 `hostID`，查库验证主机存在且 `status == "running"`
   - 解析 `multipart/form-data`，限制上传大小 100MB：`r.ParseMultipartForm(100 << 20)`
   - 从 form 中取文件字段名 `file`
   - 容器名：`"cloudproxy-" + hostID`
   - 执行：`docker exec -i containerName tar xzf - -C /`
   - 将上传文件的 `io.Reader` 直接 pipe 到 `cmd.Stdin`
   - 等待命令完成，检查 error
   - 记录事件：`admin.host.config_imported`
   - 成功返回 `{"status": "ok"}`
   - 错误处理：非 running 返回 409；无文件或文件过大返回 400；docker exec 失败返回 502

注意：
- 导入时 tar 在容器内以 `/` 为根目录解压，因为归档内的路径已经是绝对路径（`/workspace/...` 和 `/var/lib/claude-persist/...`）
- 导出时 tar 命令需要正确构造，确保归档内包含完整路径
- 两个方法都使用 `context.WithTimeout(r.Context(), 30*time.Second)`（比读写 settings 的 10 秒更长，因为归档可能较大）
- 保持与现有 `GetClaudeSettings` / `UpdateClaudeSettings` 的错误处理和日志风格一致

  <name>Task 2: 注册路由并添加前端 hooks</name>
  <files>internal/controlplane/http/router.go, web/admin/src/hooks/use-hosts.ts</files>
  <action>
1. 在 `router.go` 的 `deps.AdminHosts != nil` 块中，现有 claude 路由附近添加两行：
   ```go
   mux.Handle("GET /v1/admin/hosts/{hostID}/config/export", adminGuard(hostsHandler.ExportConfig()))
   mux.Handle("POST /v1/admin/hosts/{hostID}/config/import", adminGuard(hostsHandler.ImportConfig()))
   ```

2. 在 `use-hosts.ts` 末尾添加两个 hook：

   `useExportHostConfig()`:
   - 返回一个 mutationFn，接受 `hostId: string`
   - 使用 `fetch`（不是 `apiFetch`）直接请求，因为需要处理 blob 下载
   - 构造 URL：`/v1/admin/hosts/${hostId}/config/export`
   - 从 `localStorage.getItem("token")` 获取 token 加到 `Authorization: Bearer` header
   - 响应为 blob，创建临时 `<a>` 标签触发下载，文件名从 `Content-Disposition` 解析或默认 `host-${hostId}-config.tar.gz`
   - 错误时抛出 Error，让调用方用 toast 显示

   `useImportHostConfig()`:
   - 返回一个 mutationFn，接受 `{ hostId: string; file: File }`
   - 构造 `FormData`，字段名 `file`
   - 使用 `fetch` POST 到 `/v1/admin/hosts/${hostId}/config/import`
   - 同样从 localStorage 获取 token
   - 非 200 响应时读取 JSON error 并抛出
   - `onSuccess` 时 toast.success("配置导入成功")
