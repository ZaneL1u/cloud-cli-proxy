## Tasks

### Task 1 — 依赖与配置模块

**Actions:**
- 添加 `cobra`、`gopkg.in/yaml.v3`、`golang.org/x/term` 等依赖；将 `golang.org/x/crypto` 与仓库其余模块对齐（per STATE / D-02）。
- 实现配置结构体与 `~/.cloud-claude/` 路径解析、`config.yaml` 读写、权限 0700/0600（D-03）。
- 实现 `cloud-claude init`：交互式读取 gateway、short_id、密码（term.ReadPassword）；支持 env/flag 非交互路径（D-05）。

**Verify:** 运行 `init` 后磁盘上生成预期 YAML；错误权限或路径可测。

**Implements:** D-02, D-03, D-05, D-12

---

### Task 2 — Entry 认证与就绪轮询

**Actions:**
- 使用 `net/http` 构造 `POST {gateway}/v1/entry/{shortId}/auth`，解析 JSON（D-06）。
- 处理 `status: ready` vs `not_ready` vs HTTP 错误；实现间隔轮询与总超时（D-07）；可选 gateway 可达性预检（D-08， discretion）。
- 所有用户可见错误使用中文（D-11）。

**Verify:** 可用 mock server 或本地控制面测 ready / not_ready / 401。

**Implements:** D-06, D-07, D-08, D-11（错误文案部分）

---

### Task 3 — SSH 会话与根命令

**Actions:**
- 使用 `ssh.ClientConfig` 密码认证连接 `ssh_host:ssh_port`（D-09）；Session 申请 PTY（`x/term` 获取本地 tty 尺寸若可）。
- 在 session 上执行远程命令启动 Claude Code（例如 `claude` 或文档约定路径），无额外参数（D-10）。
- 根命令无参数时：读配置 → Task 2 → Task 3；失败时按约定退出码退出（D-11）。

**Verify:** 在主机 running 的测试环境可进入交互（或 `-e` 模拟）；错误路径返回 1–5。

**Implements:** D-01, D-09, D-10, D-11（退出码）
