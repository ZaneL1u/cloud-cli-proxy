# Phase 46 Plan 03: DNS 强制走 tun 或被防火墙拒绝 e2e

## Goal

在 worker 容器内对公共域名（如 `cloudflare.com`）发起 DNS 查询，断言任一成立即 PASS：

1. ✅ DNS 被 tun 接管，返回正常 A 记录 / TCP 握手成功；
2. ❌ DNS / 直连被防火墙明确拒绝（timeout / `Connection refused` / `Network is unreachable`）。

杜绝「DNS 走宿主机直连绕过 tun 但表现正常」这种泄漏漏判：OR 语义两端都覆盖。

## Files to touch

新增（均带 `//go:build e2e && linux`）：

- `tests/e2e/dns_test.go` — `TestDNS_TunOrFirewallDeny` 主用例。
- `tests/e2e/helpers.go` — 追加 `DNSProbeResult` 枚举 + `ClassifyDNSResult(exitCode int, stderr string) DNSProbeResult` 纯函数（`Tunneled` / `Denied` / `Leaked`）。
- `tests/e2e/helpers_test.go` — 纯函数单测：
  - `TestClassifyDNS_TunneledExitZero`
  - `TestClassifyDNS_DeniedConnectionRefused`
  - `TestClassifyDNS_DeniedTimeout`
  - `TestClassifyDNS_LeakedDirectAnswer`（一个反例：exit 0 但 stderr 出现 `bypass` 等关键字 → 视为 leaked）

不动 `go.mod`。

## Steps

1. **`ClassifyDNSResult` 纯函数**

   ```go
   type DNSProbeResult int
   const (
       DNSResultUnknown DNSProbeResult = iota
       DNSResultTunneled
       DNSResultDenied
       DNSResultLeaked
   )

   // exitCode + stderr/stdout 文案二维映射：
   //   exit 0 → Tunneled
   //   exit != 0 且 stderr 含 (connection refused|timed out|network is unreachable|operation not permitted) → Denied
   //   其它 → Unknown（用例 fail）
   //   Leaked 由调用方在拿到 ip 走宿主机的硬证据时另行赋值；ClassifyDNSResult 本身不直接判 Leaked。
   func ClassifyDNSResult(exitCode int, stderr string) DNSProbeResult
   ```

   测试覆盖 4 个分支。

2. **主用例 `TestDNS_TunOrFirewallDeny`**
   - `g := StartGoldenPath(t)`；选定测试域名（`cloudflare.com`，避免 example.com 触发 IANA 测试响应）。
   - 在 worker 容器内执行 `getent hosts cloudflare.com` 或 `dig +short @1.1.1.1 cloudflare.com`（容器镜像若不带 dig 则用 `nslookup` 兜底；优先用 `getent` 因 alpine 默认带 musl 实现）。
   - 拿到 exit code + stderr，调 `ClassifyDNSResult`：
     - `Tunneled` → 进一步 `curl --max-time 5 https://cloudflare.com -o /dev/null -w "%{http_code}"`，期望 200/301/302（HTTPS 握手成功即认 tun 接管 OK）；
     - `Denied` → 直接 PASS（防火墙生效）；
     - `Unknown` → `t.Fatal("dns probe inconclusive: ...")`。
   - **dump 钩子**：失败 / inconclusive 时调 `s.ArtifactDumper().Collect`，把 `nft list ruleset` counters 写入 `<dir>/network/dns-counters.txt`（Phase 52 OBS-02 真实接入前先占位写「计数器 dump deferred-to-OBS-02」）。

3. **避免外网抖动 false fail**
   - `cloudflare.com` 解析失败但 stderr 出现 `permanent failure in name resolution` 也视为 Denied（防火墙阻拦了上游解析器）。
   - 用例总超时 30s（`WithTimeout(30*time.Second)`）。

4. **避免 leaked false positive**
   - 如果 `Tunneled` 且后续 HTTPS 握手返回的 cert SAN 不含 cloudflare → 仍认为 Tunneled（不在本 plan 范围；leak 强对抗在 Phase 49 落地）。

## Acceptance

## Risk & Mitigation

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| `cloudflare.com` 偶发被 CI 出口屏蔽 | 低 | 用例 Skip | OR 语义 — Denied 也 PASS，本身就容错 |
| 容器镜像缺 dig / nslookup | 中 | 解析失败 | 优先 `getent hosts` + alpine busybox `nslookup` 兜底 |
| stderr 文案与 musl 版本不同 | 中 | 分类错误 | 关键字用 lower-case + `strings.Contains` 多关键字 OR；新增 case 在 helpers_test 补单测 |
| Phase 49 提前接管 DNS leak 强对抗 → 本 plan 范围漂移 | 低 | 范围重叠 | 本 plan 严格保持 OR 语义；Leaked 分类由 49-XX 单独验证 |
