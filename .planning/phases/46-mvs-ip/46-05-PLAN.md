# Phase 46 Plan 05: CLI 错误码契约 e2e

## Goal

锁定面向用户的 bootstrap 入口在 4 类错误场景下的退出码 + stderr 关键字契约，杜绝后续重构悄悄漂移：

| 触发场景 | error_code | exit code | stderr 关键字（Contains） |
|----------|-----------|-----------|---------------------------|
| 错密码 | `auth_invalid` | 10 | `用户名或密码错误` |
| 禁用账号 | `account_disabled` | 11 | `账号已被停用` |
| 过期账号 | `account_expired` | 12 | `账号已过期` |
| 无可用 host | `host_not_found` | 13 | `未找到可用主机` |

## 与 ROADMAP 的偏差说明

ROADMAP §Phase 46 §Details 5 描述 "CLI 错误码用真实 cloud-claude binary 触发各场景"。grep 现状：

- `cmd/cloud-claude/main.go` 实际定义的常量是 `exitAuthFailed=1 / exitNetworkError=2 / exitTimeout=3 / exitConfigError=4 / exitInternalError=5`，**不包含 10-13**；
- 错误码 10-13 由 `internal/controlplane/http/bootstrap_errors.go` `BootstrapErrorEntries` map 定义，并由 `deploy/bootstrap/cloud-bootstrap.sh` 在 `case "$error_code"` 分支映射到 exit code；
- 用户主入口（`curl -sSL .../v1/bootstrap/script | bash`）走的就是 bootstrap.sh，不是 cloud-claude binary。

→ **按 CONTEXT §Area 3「以源码为准」原则**：本 plan 把「真实 cloud-claude binary」换成「真实 `deploy/bootstrap/cloud-bootstrap.sh` 脚本 + 真实控制面 HTTP API」。SUMMARY 中明确记录此偏差并把它列到 VERIFICATION human-needed 项中。

## Steps

1. **fixture SQL**
   - `tests/e2e/fixtures/error-codes.sql`：
     ```sql
     -- 隐私安全：占位用户名 / bcrypt hash，禁真实邮箱
     INSERT INTO users (id, username, password_hash, status, created_at, expires_at)
     VALUES
       ('u-disabled', 'disabled-user', '$2a$10$placeholderhashforpw1234567890abc', 'disabled', NOW(), NOW() + INTERVAL '30 day'),
       ('u-expired',  'expired-user',  '$2a$10$placeholderhashforpw1234567890abc', 'active',   NOW(), NOW() - INTERVAL '1 day')
     ON CONFLICT (username) DO NOTHING;
     ```
   - 注意：真实 schema 可能字段名不同；落地时按 `internal/store/repository/` 实际列名调整，**SUMMARY 中记录**。
   - 真实密码用同一占位 bcrypt hash（明文为 `secret-placeholder-pw`），让 `account_disabled` 用例可以靠正确密码命中 disabled 分支（而非 auth_invalid 分支）。

2. **`RunBootstrapScript` helper**
   - 用 `exec.CommandContext`，`script` 是 `g.BootstrapScript` 路径，环境变量 `BOOTSTRAP_API=g.ControlPlaneURL` / `POLL_TIMEOUT=5` / `POLL_INTERVAL=1`（错误码场景不需要长 poll）。
   - stdin 喂 `<username>\n<password>\n`。
   - 收集 stdout / stderr / exitCode；`*exec.ExitError` 解包出 exit code。

3. **table-driven 主用例**

   ```go
   cases := []CLIErrorCase{
       {"auth_invalid",        "alice",          "wrong-password",      "",     10, "用户名或密码错误"},
       {"account_disabled",    "disabled-user",  "secret-placeholder-pw","",    11, "账号已被停用"},
       {"account_expired",     "expired-user",   "secret-placeholder-pw","",    12, "账号已过期"},
       {"host_not_found",      "user-no-host",   "secret-placeholder-pw","",    13, "未找到可用主机"},
   }
   for _, tc := range cases {
       s.Run(tc.Name, func() {
           code, stdout, stderr, _ := RunBootstrapScript(ctx, g.BootstrapScript, env, fmt.Sprintf("%s\n%s\n", tc.Username, tc.Password))
           s.Require().Equal(tc.WantExitCode, code, "exit code mismatch; stdout=%s stderr=%s", stdout, stderr)
           s.Require().True(strings.Contains(stdout+stderr, tc.WantStderrContains), "expected substring %q not found in output", tc.WantStderrContains)
       })
   }
   ```

4. **`host_not_found` 用例特殊处理**
   - 控制面 `BootstrapAuthHandler` 在 `GetPrimaryHostByUserID` 返回 `pgx.ErrNoRows` 时回 `host_not_found`。
   - fixture：预置一个 `active` 用户但**不**给他创建 host（与 default GoldenPath 的 alice/alpha 关系隔离 —— 用单独用户名如 `user-no-host`）。
   - 在 helpers 中加 SQL：`INSERT INTO users (...,'user-no-host',...,'active',...);`，**不**插对应 host 行。

5. **锁定表防漂移**
   - `helpers.go` 内：
     ```go
     // BootstrapExitCodeContract 与 internal/controlplane/http.BootstrapErrorEntries 对齐。
     // 编译期 import 保证源码漂移时本表也连带失败。
     var BootstrapExitCodeContract = map[string]int{
         "auth_invalid":     bootstraperrors.BootstrapErrorEntries["auth_invalid"].ExitCode,
         "account_disabled": bootstraperrors.BootstrapErrorEntries["account_disabled"].ExitCode,
         "account_expired":  bootstraperrors.BootstrapErrorEntries["account_expired"].ExitCode,
         "host_not_found":   bootstraperrors.BootstrapErrorEntries["host_not_found"].ExitCode,
     }
     ```
     但要避免 helpers.go 因 `//go:build e2e && linux` 拖累 helpers_test.go 的 darwin 编译。
   - **方案**：把 `BootstrapExitCodeContract` 放在 helpers_test.go **同包 但无 build tag** 的 helper 里也行；最稳妥是把这个变量定义在 `helpers.go`（带 tag），单测断言走「字面量预期 = 10/11/12/13」并 import `internal/controlplane/http` 拿 source-of-truth 比对（`helpers_test.go` 无 tag，可以 import 主仓 internal 包）。
   - **最终选型**：helpers_test.go 直接 import `bootstraperrors "github.com/zanel1u/cloud-cli-proxy/internal/controlplane/http"`，断言 `BootstrapErrorEntries["auth_invalid"].ExitCode == 10` 等 4 条；同时断言 `len(BootstrapErrorEntries) >= 7`（防止误删条目）。
   - 这样 darwin 上也能跑这条 lock-in 测试，不依赖 e2e fixture。

## Risk & Mitigation

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| `users` 表 schema 字段名与 SQL fixture 假设不一致 | 高 | SQL 执行失败 | 落地时先 `rg "CREATE TABLE users\|users\." internal/store/migrations/` 校对；SUMMARY 中记录实际字段 |
| bcrypt 占位 hash 与明文不匹配 | 高 | 单测 false fail | 用 `bcrypt.GenerateFromPassword([]byte("secret-placeholder-pw"), 10)` 一次性生成 hash 落到 SQL（在 SUMMARY 中标注生成方法）；不写真实凭据 |
| ROADMAP 与源码差异引发 reviewer 困惑 | 中 | review 阻塞 | PLAN / SUMMARY / VERIFICATION 三处明确记录偏差与决策依据 |
| `host_not_found` 用例与 GoldenPath 默认 fixture 冲突 | 中 | 多用例互相污染 | 用独立 username 命名空间（`user-no-host` 不在 GoldenPath alice/alpha 里） |

## 给后续 plan 的接口契约

- `BootstrapExitCodeContract` 是只读锁定表；47-XX / 48-XX 用例不应改它，要扩 error_code 必须在 `internal/controlplane/http/bootstrap_errors.go` 同步。
- `RunBootstrapScript` helper 可被 46-01 / 46-04 等场景复用作为「跑 bootstrap.sh 子进程」标准方式。
- `CLIErrorCase` 表是 e2e 内部 fixture，不外暴露。
