# Phase 46 Plan 04: 默认拒绝矩阵 e2e

## Goal

在 worker 容器内尝试直连一组「明确不该走宿主机外网」的 IP × 端口组合，断言：**任意一个连通即 fail**；全部超时 / refused / unreachable 即 PASS。

固定矩阵（CONTEXT §Area 2 锁定）：

| Host | Port | 选择理由 |
|------|------|----------|
| 1.1.1.1 | 80 | Cloudflare DNS-over-HTTP，常见旁路靶点 |
| 8.8.8.8 | 443 | Google DNS HTTPS 端口 |
| 9.9.9.9 | 443 | Quad9 DNS HTTPS 端口 |
| 169.254.169.254 | 80 | 元数据服务（云厂商 IMDS），强约束必拒 |

单次 connect 超时 3s；矩阵 4 个 target 并发探测，总用例超时 15s。

## Files to touch

新增（均带 `//go:build e2e && linux`）：

- `tests/e2e/default_deny_test.go` — `TestDefaultDeny_Matrix` 主用例。
- `tests/e2e/helpers.go` — 追加：
  - 常量切片 `DefaultDenyMatrix = []DenyTarget{...}`
  - `type DenyTarget struct{ Host string; Port int }`
  - 纯函数 `BuildDenyProbeCmd(t DenyTarget, timeoutSec int) []string` —— 返回容器内执行的 shell 命令（如 `bash -c 'echo >/dev/tcp/1.1.1.1/80'` 加 `timeout 3` 包装）。
  - 纯函数 `SummarizeDenyResults(results map[DenyTarget]int) (allDenied bool, leaks []DenyTarget)`，输入是 `{target: exitCode}`，exit 0 视为 leak。
- `tests/e2e/helpers_test.go` — 纯函数单测：
  - `TestDefaultDenyMatrix_Contents`（断言 4 行 IP/Port 与契约一致）
  - `TestBuildDenyProbeCmd_TimeoutWrapping`（断言生成的命令含 `timeout 3`）
  - `TestSummarizeDenyResults_AllDenied`
  - `TestSummarizeDenyResults_OneLeak`

## Steps

1. **常量与纯函数**

   ```go
   type DenyTarget struct {
       Host string
       Port int
   }

   var DefaultDenyMatrix = []DenyTarget{
       {"1.1.1.1", 80},
       {"8.8.8.8", 443},
       {"9.9.9.9", 443},
       {"169.254.169.254", 80},
   }

   // BuildDenyProbeCmd 返回容器内执行的 shell 命令切片。
   // 用 timeout(1) 把 bash /dev/tcp 探测裹一层硬超时（避免 nft drop 触发 TCP 长尾 retransmit）。
   func BuildDenyProbeCmd(t DenyTarget, timeoutSec int) []string {
       return []string{"timeout", strconv.Itoa(timeoutSec), "bash", "-c",
           fmt.Sprintf("echo >/dev/tcp/%s/%d", t.Host, t.Port)}
   }

   // SummarizeDenyResults 把矩阵结果合成裁决。exit 0 = 连通 = leak。
   func SummarizeDenyResults(results map[DenyTarget]int) (allDenied bool, leaks []DenyTarget)
   ```

2. **主用例**
   - `g := StartGoldenPath(t)`；从 `g.Host` 拿 worker 容器句柄。
   - 用 `errgroup` 并发对 4 target 跑 `container.Exec(ctx, BuildDenyProbeCmd(target, 3))`，单次 ctx timeout=5s，矩阵总 ctx timeout=15s。
   - 收集 `{target: exitCode}` 调 `SummarizeDenyResults`。
   - `require.True(allDenied, "leaks: %v", leaks)`。
   - 失败时 `ArtifactDumper.Collect`，在 `<dir>/network/deny-matrix.txt` 写入每个 target 的 exit code（Phase 52 OBS-02 接 nft counters）。

3. **避免 false PASS**
   - exit 0 是「连通成功」，必须 fail；
   - exit 124（GNU `timeout` 超时）/ 1 / 142 等都视为 Denied；
   - exit -1 / 错误本身 → 视为 Denied 但记 `t.Log` 提示。

4. **darwin 兜底**
   - 文件首行 `//go:build e2e && linux`；
   - helpers_test 不带 tag，darwin 上跑纯函数单测。

## Acceptance

- `go build ./tests/e2e/...`（darwin）exit 0。
- `go test ./tests/e2e/ -run "DefaultDeny|SummarizeDeny|BuildDenyProbeCmd" -count=1`（darwin）PASS。
- 单测覆盖：常量内容、命令拼装、全 Denied、单 leak。
- 无裸 `time.Sleep`。
- Linux 真机矩阵真实跑通列 deferred-to-CI。

## Risk & Mitigation

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| busybox bash 无 `/dev/tcp` | 中 | 命令直接报错 | worker 镜像默认 alpine + bash；若 busybox 兜底，用 `nc -z -w 3 host port` 替代（在 `BuildDenyProbeCmd` 内部检测 / 返回备选命令；本 plan 先选 `bash /dev/tcp` 单方案，CI 上不通再补单测） |
| 169.254.169.254 在 CI runner 上能联通（GitHub-hosted IMDS） | 高（!）| 用例 false leak | 改成 `169.254.169.254:80` 仍是 IMDS，但本 plan 严格断言「不该通」；如果 CI runner IMDS 可达，视为 host 网络隔离不严，照样 fail —— SUMMARY 中记录可能要把这一 target 改成更纯净的「闭塞地址」 |
| nft drop 触发 TCP 长尾，超过 3s | 中 | 单 probe 超时 → 矩阵总超时 | `timeout(1)` 硬截断 + 矩阵并发 errgroup |
| exit code 数值在不同 GNU/musl coreutils 版本差异 | 低 | 误判 | `SummarizeDenyResults` 只看 exit 0 vs 非 0；非 0 全部归 Denied |

## 给后续 plan 的接口契约

- `DefaultDenyMatrix` 常量被 Phase 48（Kill-switch）/ Phase 49（防泄漏）复用作为 baseline；改动需先评估冲击。
- `BuildDenyProbeCmd` 暴露给后续 plan 复用同一探测命令拼装。
