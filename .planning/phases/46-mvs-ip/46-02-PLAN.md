# Phase 46 Plan 02: 出口 IP 三源轮询 + 多数派裁决 e2e

## Goal

证明：在黄金路径起好的 worker 容器内部发起 HTTP，对应的出口 IP 与绑定的出口 IP 一致。为了避免外网抖动 / 单源故障误报，引入三源并行回显 + 多数派裁决：≥ 2 源一致即 PASS；某源 timeout / DNS 失败时视为「弃权」，不直接 fail。

## Files to touch

新增（均带 `//go:build e2e && linux`，除 helpers_test 部分）：

- `tests/e2e/egress_ip_test.go` — `TestEgressIP_MajorityVote` 主用例。
- `tests/e2e/helpers.go` — 追加 `Vote(results []string) (winner string, ok bool, dissent []string)` 纯函数；以及 `FetchEgressIPInContainer(ctx, host, sources []string)` 容器内 curl + 多源并行拉取的工具函数。
- `tests/e2e/helpers_test.go` — `TestVote_Majority`、`TestVote_AllDissent`、`TestVote_AllTimeout`、`TestVote_Tie` 等纯函数单测（无 build tag）。

不动 `go.mod`、不引入新依赖。

## Steps

1. **`Vote` 纯函数**

   ```go
   // Vote 把多源回显结果合成一个多数派裁决。
   //   - results 中空字符串视为「弃权」（该源 timeout/dns fail）。
   //   - winner 为出现次数最多的非空 IP；ok=true 当且仅当 winner 计数 >= 2。
   //   - dissent 为与 winner 不一致的非空回显（用于 artifact dump）。
   //   - 全部弃权 → winner="", ok=false。
   func Vote(results []string) (winner string, ok bool, dissent []string)
   ```

   语义对齐 CONTEXT §Area 2：「外网抖动不误报」。

2. **`FetchEgressIPInContainer`**
   - 入参：`ctx`、`harness.ContainerHandle`（worker 容器）、`sources []string`（3 个 URL）。
   - 并行 goroutine 调 `container.Exec(ctx, []string{"curl", "-fsS", "--max-time", "5", src})`，单源 5s 超时；总超时 15s。
   - 解析 stdout 拿 IP（`strings.TrimSpace`，简单 regex 校验形如 IPv4 即可）。
   - 返回 `[]string{ip1, ip2, ip3}`，超时 / 非 200 / 空响应回空字符串。

3. **主用例 `TestEgressIP_MajorityVote`**
   - `g := StartGoldenPath(t)`；从控制面 admin API 拿到 host 当前绑定的 egress IP（或直接从 Scenario 的 `GatewayHandle.EgressIP` 暴露字段，**Plan 02 需要把这个字段暴露上来**）。
   - 在 worker 容器内调 `FetchEgressIPInContainer(ctx, container, []string{"https://ip.me", "https://ifconfig.io", "https://ipinfo.io/ip"})`。
   - `winner, ok, dissent := Vote(results)`。
   - 期望：`ok == true` 且 `winner == expectedEgressIP`；`dissent` 若非空记 `t.Log` 但不 fail（用于排障）。
   - 如果 `ok == false`（全部源都失败），调 `s.ArtifactDumper().Collect(...)` 后 `t.Skip("三源全部不可达，疑似 CI 出网受限，跳过；详见 artifact")`，避免外网抖动 false-fail。

4. **artifact 钩子**
   - 失败时通过 `s.SetArtifactDumper` 自动调 `Collect`，把 `nft list ruleset` counters dump 进 `<dir>/network/`。

5. **CONTEXT 三源 URL**
   - 固定 `https://ip.me` / `https://ifconfig.io` / `https://ipinfo.io/ip`，避免每个用例自由发挥。
   - 单源 5s timeout / 总 15s timeout 用 `WaitForExec` 的 `WithTimeout(15*time.Second)` 控制。

## Acceptance

- `go build ./tests/e2e/...`（darwin）exit 0。
- `go test ./tests/e2e/ -run Vote -count=1`（darwin）PASS，至少覆盖：
  - 多数派达成（`{"1.2.3.4","1.2.3.4","5.6.7.8"}` → ok=true winner=1.2.3.4 dissent=[5.6.7.8]）
  - 全部弃权（`{"","",""}` → ok=false）
  - 平票或全部不同（`{"a","b","c"}` → ok=false dissent 长度≥2）
  - 全部一致（`{"a","a","a"}` → ok=true dissent 空）
- 文件首行 `//go:build e2e && linux`（除 `helpers_test.go` 外）。
- 无 `time.Sleep(`；lint-no-bare-sleep 通过。
- Linux 真机断言（三源真实拉取）作为 deferred-to-CI。

## Risk & Mitigation

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 三源同时被 CI runner 出口防火墙拦截 | 低 | 用例 `Skip` 太多 | `t.Skip` 而非 `t.Fail`；在 SUMMARY 中记录已知 CI 网络约束 |
| ipinfo.io 等返回 HTML / JSON 而非纯 IP | 中 | 解析错误 | `regexp.MustCompile('\\b(\\d{1,3}\\.){3}\\d{1,3}\\b')` 抠出第一个 IPv4 |
| 多数派裁决在 dissent 全部不同的 3 元结果下歧义 | 低 | 单测覆盖 | `Vote` 显式定义 winner 最多计数 < 2 时 ok=false |
| Scenario 句柄没有暴露 egress IP | 中 | 用例没法 ground truth | 在 `harness.GatewayHandle` 加 `EgressIP string` 字段（向后兼容）；SUMMARY 中说明 |
