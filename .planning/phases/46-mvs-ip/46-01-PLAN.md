# Phase 46 Plan 01: bootstrap 黄金路径 e2e

## Goal

把生产路径上「`curl https://gateway/v1/bootstrap/script | bash` → 认证 → 容器启动 → SSH banner」这条主线用一个 e2e 用例自动跑通，作为 MVS-01 黄金路径不变量的真实证据。失败一次就阻塞合并；成功一次就证明 Phase 1-44 累积的控制面 / host-agent / sing-box / Postgres 拓扑在同机串起来仍能跑通。

不在本 plan 范围：DNS / 出口 IP / 默认拒绝 / 错误码（分别由 46-02..05 覆盖）；本 plan 只关心「成功路径返回正确的 task / handoff / SSH banner」。

## Files to touch

新增（均带 `//go:build e2e && linux` build tag，除 helpers_test 外）：

- `tests/e2e/helpers.go` — 跨 plan 复用的 `GoldenPath` 抽象（在 Plan 01 里首次落地，46-02..05 import 复用）。
- `tests/e2e/helpers_test.go` — 纯函数单测，**无 build tag**（darwin 本机参与编译）；本 plan 留 `TestSanitizeHostSlug`（或类似 helper）的占位单测。
- `tests/e2e/bootstrap_test.go` — `TestBootstrap_GoldenPath` 主用例。

可能改动（仅在硬性需要时）：

- `tests/e2e/harness/scenario.go` — 若 Plan 02 的 `Start` Step 2..7 仍是 sentinel，本 plan 必须把它们实现为真实代码（控制面子进程 / admin login / fixture 三件套 / `PrepareGateway` / `PrepareHost`），并删除 `ErrScenarioStepNotImplemented` 返回路径。

不动：`go.mod`（依赖沿用 Phase 45 已落地的 testcontainers / testify）；不引入新的第三方包。

## Steps

1. **抽象 `GoldenPath`**
   - 在 `tests/e2e/helpers.go` 定义：
     ```go
     type GoldenPath struct {
         Scenario        *harness.Scenario
         User            *harness.UserHandle
         Host            *harness.HostHandle
         Gateway         *harness.GatewayHandle
         ControlPlaneURL string
         BootstrapScript string // 项目相对路径 deploy/bootstrap/cloud-bootstrap.sh
     }
     func StartGoldenPath(t *testing.T) *GoldenPath
     ```
   - `StartGoldenPath` 内部调用 `harness.New(t).WithControlPlane().WithSingBoxGateway("primary", outbound).WithHost("alpha").WithUser("alice").Start(ctx)`，并把句柄填充进 `GoldenPath`。
   - 拿不到 docker / linux 资源时 `t.Skip`（沿用 Phase 45 风格）。

2. **补 Scenario Start Step 2..7**（视当前实际状态）
   - Step 2：用 `net.Listen(":0")` 抢端口 → 关闭 → 启 `exec.CommandContext(ctx, "go", "run", "./cmd/control-plane")` 子进程，环境变量包含 `DATABASE_URL=<step1 url>` / `ADMIN_USERNAME=admin` / `ADMIN_PASSWORD=admin-pw` / `ADMIN_JWT_SECRET=e2e-admin-jwt-secret-256bit` / `CONTROL_PLANE_ADDR=:<port>`。
   - 用 `harness.WaitForPort(ctx, "127.0.0.1", port)` 等监听就绪；append kill subprocess 的 cleanup。
   - Step 3：HTTP `POST /admin/login` 获取 admin JWT；调 `POST /admin/users` / `POST /admin/egress-ips` / `POST /admin/hosts` 写入 fixture（参考 `scripts/uat-bypass-fixture-up.sh`）。
   - Step 4-6：`provider := network.NewContainerProxyProvider(s.logger)` → `provider.PrepareGateway(ctx, spec)`，spec 包含 `TunnelType=Proxy` + 占位 outbound（指向本地 SOCKS stub，必要时用 `alpine:3.20 + nc` 起一个端口转发占位即可，e2e 主要验证 control-plane / host-agent / 容器启动 → SSH 银幕拼接）。
   - Step 7：`provider.PrepareHost(ctx, spec)`，等 verify 通过；append cleanup。

3. **bootstrap 主用例**
   - `TestBootstrap_GoldenPath` 调 `StartGoldenPath(t)`。
   - 用 `exec.CommandContext(ctx, "bash", g.BootstrapScript)`，环境变量 `BOOTSTRAP_API=<g.ControlPlaneURL>` + `POLL_TIMEOUT=60` + `POLL_INTERVAL=2`，stdin 喂 `<username>\n<password>\n`。
   - 期望：
     - 子进程在 `WaitForExec`/`WaitFor` 限定时间内进入 `exec ssh` 阶段；
     - 通过观察 stdout 出现 `认证通过，主机启动中`、`SSH-2.0-OpenSSH` banner，或在控制面 events 表查到 `host.ready`，二者任一成立即通过；
     - exit code = 0（或被 `exec ssh` 替换为远端 sshd 的退出码）。
   - **双重确认**：`require.NoError(harness.WaitFor(...))` 等待 `events` 表里 type=`host.ready` 出现；同时 stdout 抓 SSH banner（`Scanner.Scan` 找 `SSH-2.0`）。任一失败即 fail。

4. **失败时自动 dump artifact**
   - 在 SetupSuite 里 `s.SetArtifactDumper(harness.NewArtifactDumper(g.Scenario, ""))`；
   - WaitFor 调用统一带 `harness.WithDumpHook(s.ArtifactDumper())`。

5. **darwin 兜底**
   - 文件首行 `//go:build e2e && linux` 保证 darwin 上跑 `go test ./tests/e2e/...` 不编译该用例；
   - `helpers_test.go`（无 tag）继续在 darwin 上跑。

## Risk & Mitigation

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Scenario Start Step 2..7 实现工作量超出单 plan 边界 | 高 | 阻塞 46-01..05 | 把 Step 2..7 看作 46-01 必备前置；若实现过半发现接口缺口大，允许把接口契约修补到 `harness/`，在 SUMMARY 里说明 |
| `go run ./cmd/control-plane` 在 CI 上首次启动慢（编译 + migration） | 中 | 用例超时 | `Scenario.Start` 用 `WaitForHTTP(/healthz, 200)` 加 90s 超时 |
| sing-box gateway 占位 outbound 在 CI 上无外网会卡住 | 中 | gateway healthy 等不到 | 选择 `direct` 类型 outbound，或本地起 alpine SOCKS5 stub；46-01 只看 bootstrap → SSH 成功，不要求真实出网 |
| testify suite SetT panic 重现 | 低 | 用例无法编译 | 沿用 Phase 45 嵌入值类型 `harness.BaseSuite` 模式 |

## 给后续 plan 的接口契约

- `helpers.StartGoldenPath(t)` 必须是幂等可重入（每个 plan 可独立调用，不复用 fixture）。

- `GoldenPath.Host` / `GoldenPath.User` 句柄被 46-02..05 直接复用。
- 任何对 `harness/` 的接口补丁必须保持向后兼容，避免破坏 Phase 47..52 plan。
