# Phase 29.1 Plan 02: worker / runtime_service fail-fast 改造 + RecordEvent + 单测 Summary

**runtime_service.QueueHostAction、worker.buildCreateArgs、worker.syncContainerCredentials 三处空密码静默 fallback 改造为 fail-fast + runtime.entry_password_missing 审计事件，并以 worker_password_test.go 两条新单测锁定回归。**

## Accomplishments

- runtime_service.QueueHostAction：空 EntryPassword 不再 `slog.Warn` 后继续派发任务，而是写事件 + 返回 `host %s entry_password is empty; refusing to queue %s` error，从源头拒绝产出"登不进去"容器的任务
- worker.buildCreateArgs：移除 `firstNonEmpty(request.EntryPassword, "workspace")`，改为入口处空值守卫直接 return error；CONTAINER_SSH_PASSWORD 直接拼真值；createHost 走既有 `taskStateFailed` err-path
- worker.syncContainerCredentials：空 EntryPassword 不再走 chpasswd，而是写 runtime.entry_password_missing 事件后立即 return；同时把 chpasswd 调用从 `exec.CommandContext` 改为 `execInContainer` 包级 var（行为等价，但单测 fake 能捕获）
- worker_password_test.go 新增 2 条单测稳定 PASS：`TestBuildCreateArgs_EmptyEntryPassword_ReturnsError` 与 `TestSyncContainerCredentials_EmptyEntryPassword_RecordsEventNoChpasswd`，复用 ssh_inject_test.go 的 setupInjectTest / fakeWorkerRepo / hasEventType 与 worker_volume_test.go 的 minimalCreateHostRequest 工厂
- worker.go 中 `"workspace"` 字面量只剩 4 处 `firstNonEmpty(request.DefaultUser, "workspace")`，全部为 Linux 用户名 fallback；密码侧已彻底清零

## Decisions Made

- runtime_service 的 Service 结构体早已包含 `RecordEvent(context.Context, repository.RecordEventParams) (repository.Event, error)` 接口字段，**无需扩大构造签名**，直接 `s.repo.RecordEvent(...)` 即可
- 选择 `source` 取值统一用短串：`queue`（runtime_service 入队拒绝）/ `sync`（worker.syncContainerCredentials 拒绝），便于运维 grep
- syncContainerCredentials 改用 `execInContainer`：原本 `exec.CommandContext("docker","exec","-i",cn,"chpasswd")` 在容器内直接 chpasswd 接 stdin；新版 `execInContainer(ctx, cn, "chpasswd", "user:pass\n")` 等价于 `docker exec -i cn bash -c "chpasswd"`，多一层 bash 但 chpasswd 是 PATH 内常驻命令，行为完全一致；好处是单测的 fake runner 能捕获到 fc.log 字符串
- minimalCreateHostRequest 工厂默认填密码占位是必要的 Rule 3 修复，否则 `TestBuildCreateArgs_VolumesMount` / `TestBuildCreateArgs_EmptyVolumes_NoExtraArgs` / `TestBuildCreateArgs_InvalidVolumeMount` 等无关用例会全部因为新空密码守卫而 fail；占位串明显非真实密码（`pw-default-test`），不会被误读为安全凭据
