# Phase 48 Plan 01: sing-box 崩溃后用户容器立即断网 (MVS-09)

## Goal

证实「`docker kill -SIGKILL` sing-box gateway 后，用户容器在 3 秒内立即失去出网能力，并且 **不会** 回落到宿主机直连」。落地为 darwin 上可静态校验、Linux runner 上可真实跑通的 e2e 用例。

具体到不变量层面：
- 容器内 `curl --max-time 3 <egressIPSource>` 必须在 3 秒内非 0 退出。
- 通过 host eth0 抓包独立 oracle：tcpdump 在窗口内必须捕获 **0** 个「源=worker IP / 目的 != gateway IP」的包，证明没有任何流量绕过 sing-box 直接走宿主机出网。
- timing 契约：kill→断网 ≤ 3s（与 Phase 50 KILL-01 一致）。

**不在本 plan 范围**（属 Phase 50 KILL-02..04）：

- `docker network disconnect` / `ip link set tun0 down` / Pumba netem 模拟。
- sing-box 重启后流量恢复链路验证。
- 多 worker 并发 kill-switch。

## Steps

1. **grep 锁定 source-of-truth**
   - `internal/network/container_proxy_provider.go::gatewayContainerName(hostID) → "cloudproxy-gw-" + hostID`。
   - `GatewayHandle.ContainerID` 字段在 Phase 45 Step 4..6 真实落地后填充；本 plan 优先消费它，**没有**再回退到命名约定 `cloudproxy-gw-<HostID>`。
   - `internal/network/container_proxy_provider.go::workerContainerName(hostID) → "cloudproxy-" + hostID`。

2. **纯函数 `ParseTcpdumpCountOutput`**
   - 输入：tcpdump 子进程 stderr（tcpdump 把 banner 与计数都写 stderr）。
   - 行为：扫第一行匹配正则 `^(\d+) packets? captured`，命中即返回数字；缺失返回 `(0, errCountNotFound)`；空 stderr 返回 `(0, errCountNotFound)`。
   - 不做 trim/lower 处理之外的解析，避免吞掉 tcpdump 真实错误（permission denied / no such device 等）。

3. **纯函数 `ClassifyKillswitchResult`**
   - 入参：
     - `probeExitCode`：容器内 curl 退出码（0 = 仍能出网，非 0 = 已断网；timeout=124）。
     - `leakedPackets`：host eth0 tcpdump 在 bpf `src host <worker> and not dst <gateway>` 下捕获的非网关包数。
   - 输出枚举：
     - `KillswitchOK`：`probeExitCode != 0 && leakedPackets == 0`。
     - `KillswitchProbeUnexpectedlySucceeded`：`probeExitCode == 0 && leakedPackets == 0`。
     - `KillswitchPacketLeak`：`probeExitCode != 0 && leakedPackets > 0`。
     - `KillswitchBoth`：`probeExitCode == 0 && leakedPackets > 0`（最坏情况）。
   - 单测覆盖 4 分支。

4. **`KillswitchTimingContract`**
   - 字段：`ProbeMaxLatency=3s`、`TcpdumpWindow=5s`。任一漂移 → darwin 单测 fail，Phase 50 必须同步修。

5. **`GoldenPath.KillGateway`**
   - 实现：
     ```go
     containerID := p.Gateway.ContainerID
     if containerID == "" {
         containerID = fmt.Sprintf("cloudproxy-gw-%s", p.Host.ID) // Phase 45 Step 4..6 fallback
     }
     cmd := exec.CommandContext(ctx, "docker", "kill", "--signal=KILL", containerID)
     ```
   - 错误：stderr 含 `is not running` / `No such container` → 返回 wrapped error，让用例 fast-fail。
   - 不做 `docker stop`（会走 sing-box graceful shutdown），CONTEXT §Area 1 明确要求 SIGKILL。

6. **`GoldenPath.ProbeOutboundFromUser`**
   - 实现：复用 `workerContainerHandle(gp).Exec(ctx, []string{"curl","-sS","--max-time", strconv.Itoa(int(timeout.Seconds())), url})`。
   - 返回 `exitCode + err`；stdout 不暴露给用例（kill-switch 验证只看 exitCode）。
   - 容器句柄未就绪 → 返回 `errWorkerContainerHandleUnavailable`，用例 `t.Skip`。

7. **`GoldenPath.TcpdumpOnHostEth0`**
   - 实现路径选择：
     - **路径 A（首选）**：testcontainer host network sidecar。起一个 `nicolaka/netshoot:v0.13` 容器 `--network host --cap-add NET_RAW --cap-add NET_ADMIN`，命令 `tcpdump -nn -i eth0 -c <count> <bpfFilter>`，等其在 `timeout` 内退出，收 stderr 喂 `ParseTcpdumpCountOutput`。
     - **路径 B（兜底）**：`exec.CommandContext(ctx, "sudo", "tcpdump", ...)`。仅当 `os.Getuid() == 0` 或 `E2E_ALLOW_HOST_TCPDUMP=1` 才启用；否则在 darwin / 非特权 Linux runner 上 t.Skip。
   - 默认走路径 A；CI workflow 已有 `sudo apt-get install tcpdump` 与 `--cap-add NET_RAW` 模板（参考 `.github/workflows/uat-bypass.yml` 第 115 行），实现侧不引入新 systemd 依赖。
   - 接口签名稳定：`(packets int, err error)`，不暴露 sidecar 容器句柄给用例。

8. **e2e 主用例 `TestKillSwitch_SingboxCrash_GoldenPath`**
   - `StartGoldenPath(t)`；nil → return（同 Phase 46/47 范式）。
   - 总 `ctx, cancel := context.WithTimeout(ctx, 60*time.Second)`。
   - 基线：先 `g.ProbeOutboundFromUser(ctx, "https://ifconfig.io", 5s)` 必须 exitCode==0（worker 出网正常）。基线失败 → `t.Skipf("baseline egress not working: %v", err)`（与 Phase 46 范式一致，避免外网抖动 false-fail）。
   - **关键步骤**：
     1. 后台启 `TcpdumpOnHostEth0(ctx, "src host <workerIP> and not dst <gatewayIP>", 5, KillswitchTimingContract.TcpdumpWindow)` 作为 oracle（用 goroutine + channel）。
     2. `KillGateway(ctx)`。
     3. 立即（不睡眠）以 `ProbeOutboundFromUser(ctx, src, KillswitchTimingContract.ProbeMaxLatency)` 探测，期望 exitCode != 0。
     4. 等 tcpdump goroutine 退出，读 `packets` 与 `err`。
     5. `verdict := ClassifyKillswitchResult(probeExitCode, packets)`；`verdict != KillswitchOK` → `t.Fatalf`，并把 (verdict / probeExitCode / packets / curl stderr 末尾) 打到 t.Logf。
   - workerIP / gatewayIP 来源：`g.Host.ContainerName` + `docker inspect -f '{{...}}'`；用现有 `workerContainerHandle` 的 `Exec` 跑 `hostname -i` 拿 worker 容器 IP；gateway IP 直接读 `g.Gateway.GatewayIP`（Phase 45 在 Step 4..6 落地后填充，未填充 → t.Skip）。

9. **darwin 兜底**
   - `killswitch_singbox_crash_test.go` 首行 `//go:build e2e && linux`。
   - `helpers.go` 新增片段无 tag；darwin 上 `go test ./tests/e2e/ -run Helpers` 必须 PASS（覆盖 9 + 4 + 1 = 14 个新单测）。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| testcontainer host 没有 `eth0`（macOS Desktop Docker 用 LinuxKit VM，host network sidecar 在 macOS 上不抓到宿主机 NIC） | darwin 上整个 e2e 测试通过 build tag 跳过；Linux CI runner（hosted ubuntu-24.04）`eth0` 是默认 NIC，已在 Phase 45 uat-bypass.yml workflow 中验证可用。 |
| 路径 A 的 netshoot sidecar 拉镜像慢导致 tcpdump 启动晚于 curl | sidecar 起来后通过 `WaitForLog` 等到 `listening on eth0` 字符串再返回主流程（实现层）；用例侧把启 sidecar 与 KillGateway 之间留逻辑上的「先后顺序」，不靠 sleep。 |
| sing-box 容器被 SIGKILL 但 nft / tun0 残留规则仍允许出网 | 这正是 MVS-09 真要测的：tcpdump 0 包 + curl 非 0 退出 ⇔ kill-switch 兜住了。如果跑出来 `KillswitchProbeUnexpectedlySucceeded`（curl 仍出网），就是被测代码的真实回归，不是测试 bug。 |
| `docker kill` 返回 0 但容器实际还活着 | `KillGateway` 调完后调 `docker inspect -f '{{.State.Running}}'` 二次确认；非 false → 返回 error。 |
| `GatewayHandle.ContainerID` 在 Scenario Step 2..7 sentinel 期间为空 | fallback 到 `cloudproxy-gw-<HostID>`；HostID 也为空时 t.Skip（与 expiry / host_agent_recovery 用例一致）。 |
| host eth0 tcpdump 需要 root / cap_net_admin | 路径 A 起 `--cap-add NET_RAW --cap-add NET_ADMIN`；路径 B 仅在 `E2E_ALLOW_HOST_TCPDUMP=1` 时启用。两路径都失败 → t.Skip 并把原因写入 t.Logf，VERIFICATION 显式记录这是 deferred-to-CI 项。 |
