  <action>
1. 添加 shellescape 依赖：
   ```bash
   go get al.essio.dev/pkg/shellescape@latest && go mod tidy
   ```

2. 修改 `ConnectAndRunClaude` 函数签名，从：
   ```go
   func ConnectAndRunClaude(cfg SSHConfig) error
   ```
   改为：
   ```go
   func ConnectAndRunClaude(cfg SSHConfig, claudeArgs []string) (int, error)
   ```
   新增 `claudeArgs` 参数承接用户传入的 claude 参数；返回 `(exitCode int, err error)`，**禁止在函数内调用 `os.Exit`**（per D-06，修复 HI-01）。

3. 在 `session.Start` 之前构建安全的远程命令行（per D-02）：
   ```go
   import "al.essio.dev/pkg/shellescape"

   remoteCmd := shellescape.QuoteCommand(append([]string{"claude"}, claudeArgs...))
   ```
   将现有 `session.Start("claude")` 替换为 `session.Start(remoteCmd)`。

4. 重构 TTY / 非 TTY 分支（per D-08）。用 `isTTY := term.IsTerminal(fd)` 判断：
   - **TTY 路径（isTTY == true）**：保留现有逻辑——`term.GetSize` → `term.MakeRaw` + `defer term.Restore` → `session.RequestPty("xterm-256color", height, width, modes)` → SIGWINCH 监听。
   - **非 TTY 路径（isTTY == false）**：跳过 `MakeRaw`、`RequestPty` 和 SIGWINCH 监听。`session.Stdin/Stdout/Stderr` 仍连接 `os.Stdin/Stdout/Stderr`，但不申请 PTY。

5. 修复退出码处理（per D-06 / D-07）。将现有的：
   ```go
   if exitErr, ok := err.(*ssh.ExitError); ok {
       os.Exit(exitErr.ExitStatus())  // ← 跳过 defer Restore！
   }
   ```
   替换为：
   ```go
   if exitErr, ok := err.(*ssh.ExitError); ok {
       return exitErr.ExitStatus(), nil
   }
   ```
   正常退出返回 `(0, nil)`；`io.EOF` 返回 `(0, nil)`；其他错误返回 `(0, fmt.Errorf(...))` 由调用方映射到 `exitInternalError`。

6. 移除 `"os"` 包中不再使用的 `os.Exit` 引用（如果 `os` 仍被 `os.Stdin` 等使用则保留包导入）。

   rootCmd := &cobra.Command{
       Use:                "cloud-claude",
       Short:              "透明远程 Claude Code CLI",
       Long:               "连接远端云主机并启动 Claude Code 交互会话。\n首次使用请先运行 cloud-claude init 配置网关与凭证。",
