# Phase 50 Plan 01: SIGKILL gateway timing 严格化 (KILL-01)

## Goal

在 Phase 48 MVS-09 已交付「sing-box 崩溃后用户容器立即断网」基础上，**收紧 timing 契约**：
`docker kill --signal=KILL <gateway>` 后必须在 **≤ 3s** 窗口内完成「容器内 curl 必须非 0 退出 + host eth0 抓包零非网关流量」。本 plan 不复用 Phase 48 用例文件，而是新增 `tests/e2e/killswitch_stress/killswitch_01_sigkill_timing_test.go`，在同套 helper 上加 **wall-clock 耗时断言** 与 **更严格 BPF**。

具体不变量：

- `KillGateway → ProbeOutboundFromUser` 端到端耗时 ≤ 3s（含 docker exec 启动 overhead 容差）。
- host eth0 抓包 BPF `src host <workerIP> and not dst host <gatewayIP>` 在 5s 窗口内必须 0 包。
- ClassifyStressResult 合成裁决 = `StressVerdictPass`；任何 `Fail` 直接 t.Fatalf。
- 与 Phase 48 MVS-09 互不替代：MVS-09 守护「行为」，KILL-01 守护「timing + 多源断言一致性」。

**不在本 plan 范围**：

- `ip link set tun0 down`（属 KILL-02）。
- Pumba netem 注入（属 KILL-03）。
- `docker network disconnect`（属 KILL-04）。
- gateway 重启后流量恢复（属未来 phase）。

## Steps

1. **grep 锁定 source-of-truth**
   - `internal/network/container_proxy_provider.go::gatewayContainerName(hostID) → "cloudproxy-gw-" + hostID`。
   - `internal/network/container_proxy_provider.go::workerContainerName(hostID) → "cloudproxy-" + hostID`。
   - `internal/runtime/tasks/worker.go:215` worker 容器 `--network bridge`，docker 默认 bridge；KILL-01 BPF `not dst host <gatewayIP>` 中 gatewayIP 来源是 `GatewayHandle.GatewayIP`。
   - tun 设备实名：gateway 容器内 `tun0`（`waitForTun0` / `singbox_provider_linux.go:161`），worker netns oifname `sb-tun0` 是 nft 规则用的接口名（与本 plan 无关，与 KILL-02 也需区分）。

2. **`StressVerdict` 三值枚举 + `StressEvidence` 结构**
   - `Pass / Fail / Inconclusive` 三值；与 Phase 49 `LeakVerdict` 风格相同但语义独立。
   - `StressEvidence{ProbeExitCode int, LeakedPackets int, ElapsedMs int, SSHAlive bool, EgressIPVote VoteResult}`：
     - 字段 `EgressIPVote` 复用 Phase 46 `VoteResult`，KILL-03 用到；KILL-01/02/04 留 zero value。
     - 字段 `SSHAlive` KILL-03 用到；KILL-01/02/04 不读。

3. **`KillswitchStressContract` 锁定表**
   ```go
   var KillswitchStressContract = map[string]struct {
       MaxDisconnectMs   int
       SSHAlive          bool
       AllowInconclusive bool
   }{
       "KILL-01": {MaxDisconnectMs: 3000, SSHAlive: false, AllowInconclusive: false},
       "KILL-02": {MaxDisconnectMs: 3000, SSHAlive: false, AllowInconclusive: false},
       "KILL-03": {MaxDisconnectMs: 0,    SSHAlive: true,  AllowInconclusive: true},
       "KILL-04": {MaxDisconnectMs: 3000, SSHAlive: false, AllowInconclusive: false},
   }
   ```
   `MaxDisconnectMs=0` 表示 KILL-03 不做断网 timing 断言（只要求 SSH 存活）。任一漂移 → darwin 单测立即 fail。

4. **纯函数 `ClassifyStressResult(kill string, ev StressEvidence)`**
   - 行为表：
     - `kill` 未知 → `Inconclusive, "unknown kill id"`。
     - `KILL-01/02/04`（`MaxDisconnectMs > 0`）：
       - `ProbeExitCode == 0` → `Fail, "probe unexpectedly succeeded"`。
       - `LeakedPackets > 0` → `Fail, "host eth0 captured N non-gateway packets"`。
       - `ElapsedMs > MaxDisconnectMs` → `Fail, "disconnect latency NMs exceeds threshold"`。
       - 其它 → `Pass`。
     - `KILL-03`：
       - `SSHAlive == false` → `Fail, "ssh banner not received within timeout"`。
       - `EgressIPVote.OK && winner != expectedWinner` → `Fail, "vote winner X != expected"`（exposed via field）。
       - `EgressIPVote` 全弃权 + `AllowInconclusive == true` → `Inconclusive, "vote all-abstain (allowed under netem stress)"`。
       - 其它 → `Pass`。
   - 单测覆盖 ≥ 12 行为分支 + 3 锁定。

5. **e2e 主用例 `TestKillSwitch_01_SigkillTiming`**
   - `StartStressGolden(t)`；nil → return；host / gateway 句柄缺失 → t.Skip。
   - 总 `ctx, cancel := context.WithTimeout(ctx, 60*time.Second)`。
   - 基线：`g.ProbeOutboundFromUser(ctx, "https://ifconfig.io", 5s)` 必须 exit 0；否则 t.Skipf（与 Phase 48 范式一致）。
   - 解析 workerIP / gatewayIP（复用 `g.InspectContainerIPv4(ctx, workerName, "")`、`g.Gateway.GatewayIP` 兜底 `g.InspectContainerIPv4(ctx, gatewayName, "")`）。
   - **关键步骤**：
     1. 后台启 `g.TcpdumpOnHostEth0(ctx, "src host <workerIP> and not dst host <gatewayIP>", 5, KillswitchTimingContract.TcpdumpWindow)`（goroutine + channel）。
     2. `start := time.Now()`；`g.KillGateway(ctx)`；不睡眠。
     3. `probeExit, probeErr := g.ProbeOutboundFromUser(ctx, probeURL, KillswitchStressContract["KILL-01"].MaxDisconnectMs)`，记 `elapsedMs := time.Since(start).Milliseconds()`。
     4. 等 tcpdump goroutine（select ctx.Done 兜底）。
     5. `verdict, reason := ClassifyStressResult("KILL-01", StressEvidence{ProbeExitCode: probeExit, LeakedPackets: packets, ElapsedMs: int(elapsedMs)})`。
     6. `verdict != StressVerdictPass` → `t.Fatalf("KILL-01 fail: verdict=%s reason=%s elapsed=%dms probeExit=%d packets=%d", ...)`。
   - cleanup：本用例不做 gateway 恢复（与 Phase 48 一致；后续用例自带 GoldenPath 隔离）。

6. **darwin 兜底**
   - `killswitch_01_sigkill_timing_test.go` 首行 `//go:build e2e && linux`。
   - 新增纯函数与 contract 由 50-shared 提交统一引入；darwin `go test ./tests/e2e/ -run "Helpers"` 必须 PASS。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| docker exec startup overhead 抬高 elapsedMs，3s 阈值过严 | `MaxDisconnectMs` 锁 3000ms 含 ~200ms exec overhead；超阈不直接判 Fail，先 t.Logf 完整时间线，由 CI runner 真机签字时若稳定超出再放宽到 3500ms（VERIFICATION 显式记录此调整窗口）。 |
| host eth0 抓包在 hosted ubuntu-24.04 没有 root | `TcpdumpOnHostEth0` 已支持 sidecar `--cap-add NET_RAW NET_ADMIN`；不可用时 t.Skip 并 deferred-to-CI（与 Phase 48 同策略）。 |
| Phase 48 用例文件名冲突 | 本 plan 用例放 `tests/e2e/killswitch_stress/` 子目录 + 文件名前缀 `killswitch_NN_<name>`；与 Phase 48 `tests/e2e/killswitch_singbox_crash_test.go` 完全分离。 |
| gateway SIGKILL 后 docker 自动清理容器导致 InspectContainerIPv4 拿不到 IP | gatewayIP 在 KillGateway 前就解析并缓存到本地变量；之后不再 inspect。 |
