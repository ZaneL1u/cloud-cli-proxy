# Phase 50 Plan 03: Pumba netem delay → SSH 存活 + 出口 IP 允许 inconclusive (KILL-03)

## Goal

证实「在 gateway 容器上注入 Pumba netem `delay 1000ms` 后，worker 容器内 **SSH 控制流必须存活**（22 端口 nc / banner 探测成功），但 **出口 IP 多数派投票允许超时**（Phase 46 `Vote` 全弃权时不 fail）。失败约束：≥2 源回显「非预期 IP」立刻 Fail。」与 KILL-01/02/04 的「硬故障 → 立即断网」不同，KILL-03 锁定「软劣化 → 行为契约固定」的语义。

具体不变量：

- Pumba sidecar 在 gateway 容器上跑 `netem --duration 30s delay --time 1000`；用例期间持续生效。
- SSH 存活：`g.ProbeSSHBanner(ctx, 10s)` 必须返回 nil（拿到 SSH-2.0 banner 或 nc -z 成功）。
- 出口 IP：`g.FetchEgressIPInContainer(ctx, ...)` + `e2e.Vote(...)` 三源；
  - 全部 timeout → `StressVerdictInconclusive`（contract `AllowInconclusive=true` 命中）。
  - ≥ 2 源回显同一 IP 且 == expected → Pass。
  - ≥ 2 源回显同一 IP 且 ≠ expected → Fail。
- timing 维度：本 plan **不**对断网延迟做断言（`MaxDisconnectMs=0`）。
- cleanup：Pumba `--duration 30s` 自然结束；用例侧 `defer pumbaCleanup()` 触发 docker kill sidecar 兜底。

**不在本 plan 范围**：

- 验证 sing-box 在劣化下重连（属未来 phase）。
- 多 Pumba target 并行注入。
- loss 模式 + delay 叠加（CONTEXT §Specifics 留备选；本 plan 锁单参数）。

## Steps

1. **grep 锁定 source-of-truth**
   - Pumba 官方 image：`gaiaadm/pumba:0.10.0` 固定 tag（CONTEXT §Specifics「Claude's Discretion」）；避免 `latest` 漂移。
   - Pumba CLI 调用形态：`docker run --rm -v /var/run/docker.sock:/var/run/docker.sock gaiaadm/pumba:0.10.0 netem --duration <dur> --tc-image gaiadocker/iproute2 delay --time <ms> <target-name>`。
   - target 应为 gateway 容器名 `cloudproxy-gw-<HostID>`；Pumba 内部用 `docker exec --privileged <tc-image>` 注入 tc 规则。
   - SSH 端口：worker 镜像 OpenSSH 监听 22（`internal/runtime/tasks/worker.go:218 entrypoint`），darwin runtime 走 `-p 0:22` host port mapping，linux runtime 直连容器 IP:22。本用例通过 `docker exec <worker> nc -z localhost 22` 在容器内探测，跨平台路径一致。

2. **`PumbaNetemParams` + `BuildPumbaNetemArgs` 纯函数**
   - 结构：
     ```go
     type PumbaNetemParams struct {
         Mode     string        // "delay" / "loss" / "duplicate"
         DelayMs  int           // delay 模式
         LossPct  int           // loss 模式
         Duration time.Duration
         Image    string        // 默认 "gaiaadm/pumba:0.10.0"
         TcImage  string        // 默认 "gaiadocker/iproute2"
     }
     ```
   - `BuildPumbaNetemArgs(target, params)` 返回完整 `docker run` argv；空 / 非法字段走默认；`DelayMs <= 0` 默认 1000。
   - 单测覆盖 ≥ 3 case：delay 模式默认值、custom image / tc-image、target 空字符串报错路径（返回 nil）。

3. **`ParsePumbaOutput` 纯函数 + fixture 单测**
   - `PumbaOutcome` 枚举（`Unknown / Applied / Failed / ImageMissing / DaemonDown`）+ `String()`。
   - 行为：
     - stdout 含 `tc/netem` / `applying tc policy` / `successfully` → `Applied`。
     - stderr 含 `manifest unknown` / `pull access denied` / `image not found` → `ImageMissing`。
     - stderr 含 `Cannot connect to the Docker daemon` → `DaemonDown`。
     - 其它 → `Failed`。
   - 单测从 `tests/e2e/testdata/killswitch/pumba_*.txt` fixture 读取（沿用 Phase 49 `readLeakFixture` 模式，新加 `readKillFixture` helper）。
   - 单测覆盖 ≥ 4 case：applied / image_missing / daemon_down / unknown_garbage。

4. **`GoldenPath.InjectPumbaNetem`**
   - 实现路径：
     ```go
     argv := BuildPumbaNetemArgs(target, params)
     sidecarName := fmt.Sprintf("pumba-%s-%d", target, time.Now().UnixNano())
     full := append([]string{"docker", "run", "--rm", "--name", sidecarName,
         "-v", "/var/run/docker.sock:/var/run/docker.sock"}, argv[1:]...)
     cmd := exec.CommandContext(ctx, full[0], full[1:]...)
     if err := cmd.Start(); err != nil { return nil, fmt.Errorf("start pumba: %w", err) }
     cleanup := func() {
         _ = exec.Command("docker", "kill", sidecarName).Run()
         _ = cmd.Wait()
     }
     return cleanup, nil
     ```
   - Pumba `--duration` 自然结束时 cmd.Wait 自动 reap；用例侧手动 cleanup 是兜底。
   - 如果 `exec.LookPath("docker")` 失败 / image pull 错（通过短超时 + retry 0 探测），返回 err，调用方 t.Skip。

5. **`GoldenPath.ProbeSSHBanner`**
   - 实现：
     ```go
     argv := []string{"bash", "-c",
         "exec 3<>/dev/tcp/localhost/22 && head -c 6 <&3 | grep -q '^SSH-'"}
     ```
   - 通过 `docker exec <worker> bash -c '...'`，timeout 由 ctx 控制。
   - exit 0 → nil；非 0 / docker exec 错 → wrap err 返回。
   - 复用 `workerDockerName()`；句柄缺失 → `errWorkerContainerHandleUnavailable`。

6. **e2e 主用例 `TestKillSwitch_03_NetemDelay`**
   - `StartStressGolden(t)`；nil / 句柄缺失 → return / Skip。
   - 总 `ctx, cancel := context.WithTimeout(ctx, 90*time.Second)`（Pumba `--duration 30s` + 探测窗口）。
   - 前置：`exec.LookPath("docker")` 失败 → t.Skipf；CONTEXT 备选「hosted runner pull pumba image 失败」兜底逻辑。
   - 基线：`g.ProbeOutboundFromUser(ctx, "https://ifconfig.io", 5s)` 必须 exit 0；否则 t.Skipf。
   - 解析 expected egressIP（FetchEgressIPInContainer 三源，Vote 取 winner；非多数派 → t.Skipf "baseline vote not stable"）。
   - **关键步骤**：
     1. `cleanup, err := g.InjectPumbaNetem(ctx, gatewayName, PumbaNetemParams{Mode:"delay", DelayMs:1000, Duration:30*time.Second})`；
        err 含 `ImageMissing` → t.Skipf；其它 err → t.Fatalf。
        `defer cleanup()`。
     2. SSH 存活探测：`if err := g.ProbeSSHBanner(ctx, 10*time.Second); err != nil { sshAlive = false }`。
     3. 出口 IP 投票：`raw := g.FetchEgressIPInContainer(ctx, workerHandle)`；`vote := e2e.Vote(raw)`。
     4. `verdict, reason := ClassifyStressResult("KILL-03", StressEvidence{SSHAlive: sshAlive, EgressIPVote: vote, ExpectedEgressIP: expectedIP})`。
     5. `verdict == Fail` → t.Fatalf；`Inconclusive` → t.Skipf（contract `AllowInconclusive=true` 命中，记录 reason 到 SUMMARY，不算 ROADMAP 失败）；`Pass` → t.Logf + return。
   - cleanup 在 t.Cleanup 之外用 `defer cleanup()` 注册（pumba sidecar 提前 docker kill 避免拖延总耗时）。

7. **darwin 兜底**
   - 用例文件首行 `//go:build e2e && linux`。
   - `ParsePumbaOutput` + `BuildPumbaNetemArgs` 在 helpers.go 无 tag；darwin `go test ./tests/e2e/ -run "Helpers"` 必须 PASS。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| hosted ubuntu-24.04 runner pull `gaiaadm/pumba:0.10.0` 失败（rate-limit / 私网） | `ParsePumbaOutput` 识别 `ImageMissing` → 用例 t.Skipf + SUMMARY 记录 "deferred-to-self-managed-runner"；CONTEXT 备选已锁此兜底。 |
| Pumba netem 注入失败（gateway 容器内 tc 二进制缺） | Pumba `--tc-image gaiadocker/iproute2` 走外部 sidecar 注入（不依赖 gateway 容器内有无 tc）；image 不可用回到上一行 ImageMissing 分支。 |
| SSH 22 端口未启动（worker 镜像 entrypoint 慢） | 基线 SSH banner 探测在 Pumba 注入之前先做一次（隐式校验 22 已 listen）；netem 注入后再次探测，二者对比表达「劣化但不死」。基线失败 → t.Skipf。 |
| Vote 全弃权但实际是 backend 真断网 | contract `AllowInconclusive=true` 明确允许全弃权；本 plan 不区分「劣化导致 timeout」与「真断网」，因为 KILL-03 锁的是「SSH 存活 + 不允许错误回显」，不是「必须有回显」。 |
| Pumba `--duration 30s` 期间 cleanup 主动 docker kill 留 dangling tc 规则 | Pumba 在 SIGTERM 时会自动清理（其设计目标）；用例侧 `cleanup` 直接 docker kill 简单可靠，余留 tc 规则属本 phase 不展开（极端情况下后续用例 Scenario.Stop 销毁 gateway 一并清理）。 |
